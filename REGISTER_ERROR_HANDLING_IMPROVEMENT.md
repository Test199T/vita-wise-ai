# การปรับปรุงการจัดการข้อผิดพลาดในการสมัครสมาชิก

## ปัญหาที่แก้ไข
เดิมเมื่อผู้ใช้ลองสมัครสมาชิกด้วยอีเมลที่มีอยู่แล้วในระบบ จะแสดงข้อความทั่วไปว่า "เกิดข้อผิดพลาดข้อมูลไม่ถูกต้อง" ซึ่งไม่ชัดเจน

## การแก้ไขที่ทำ

### 1. ปรับปรุงข้อความแจ้งเตือนสำหรับอีเมลซ้ำ
- **เดิม**: "ข้อมูลไม่ถูกต้อง"
- **ใหม่**: "มีผู้ใช้อีเมลนี้แล้ว กรุณาใช้อีเมลอื่น"

### 2. เพิ่มการตรวจสอบข้อผิดพลาดที่เฉพาะเจาะจง

#### Status Code 400 (Bad Request)
- ตรวจสอบข้อความจาก backend เพื่อหากรณีอีเมลซ้ำ
- แสดงข้อความที่เหมาะสม:
  - "มีผู้ใช้อีเมลนี้แล้ว กรุณาใช้อีเมลอื่น" (กรณีอีเมลซ้ำ)
  - "มีบัญชีนี้แล้ว กรุณาใช้อีเมลอื่น" (กรณีบัญชีซ้ำ)
  - "ข้อมูลไม่ถูกต้อง กรุณาตรวจสอบข้อมูลที่กรอก" (กรณีอื่นๆ)

#### Status Code 409 (Conflict)
- แสดงข้อความ: "มีผู้ใช้อีเมลนี้แล้ว กรุณาใช้อีเมลอื่น"

#### Status Code 422 (Unprocessable Entity)
- ตรวจสอบ validation errors ที่เฉพาะเจาะจง
- แยกแยะระหว่างอีเมลซ้ำและรูปแบบอีเมลไม่ถูกต้อง

#### Status Code 500 (Internal Server Error)
- แสดงข้อความ: "เกิดข้อผิดพลาดที่เซิร์ฟเวอร์ กรุณาลองใหม่อีกครั้ง"

#### กรณีอื่นๆ
- ตรวจสอบข้อความจาก backend เพื่อหากรณีอีเมลซ้ำ
- แสดงข้อความที่เหมาะสมตามประเภทข้อผิดพลาด

### 3. ปรับปรุงการจัดการข้อผิดพลาดในส่วน catch block
- แยกแยะประเภทข้อผิดพลาดการเชื่อมต่อ
- แสดงข้อความที่เฉพาะเจาะจงสำหรับแต่ละกรณี

## ตัวอย่างข้อความแจ้งเตือนใหม่

### กรณีอีเมลซ้ำ
- "มีผู้ใช้อีเมลนี้แล้ว กรุณาใช้อีเมลอื่น"
- "มีบัญชีนี้แล้ว กรุณาใช้อีเมลอื่น"

### กรณีข้อมูลไม่ถูกต้อง
- "ข้อมูลไม่ถูกต้อง กรุณาตรวจสอบข้อมูลที่กรอก"
- "รูปแบบอีเมลไม่ถูกต้อง"

### กรณีการเชื่อมต่อ
- "ไม่สามารถเชื่อมต่อกับเซิร์ฟเวอร์ได้ กรุณาตรวจสอบการเชื่อมต่ออินเทอร์เน็ต"
- "การเชื่อมต่อใช้เวลานานเกินไป กรุณาลองใหม่อีกครั้ง"
- "เกิดข้อผิดพลาดในการเชื่อมต่อ กรุณาลองใหม่อีกครั้ง"

## ผลลัพธ์ที่คาดหวัง
ผู้ใช้จะเห็นข้อความแจ้งเตือนที่ชัดเจนและเข้าใจง่าย เมื่อลองสมัครสมาชิกด้วยอีเมลที่มีอยู่แล้วในระบบ แทนที่จะเห็นข้อความทั่วไปที่ไม่ชัดเจน

## การทดสอบ
1. ลองสมัครสมาชิกด้วยอีเมลที่มีอยู่แล้วในระบบ
2. ตรวจสอบว่าข้อความแจ้งเตือนแสดง "มีผู้ใช้อีเมลนี้แล้ว กรุณาใช้อีเมลอื่น"
3. ทดสอบกรณีข้อผิดพลาดอื่นๆ เพื่อให้แน่ใจว่าข้อความแสดงถูกต้อง
